# 沙丘色 HTML PPT 设计规范

这套风格面向发布会、主题演讲、产品发布和高端宣讲。核心气质是：接近纯色的奶油沙丘背景、陶土橙强调、极粗大黑标题、强留白、强舞台感。它不是传统汇报 PPT，也不是网页落地页；每一页都应该像发布会大屏上的一个关键时刻。

## 1. 视觉方向

关键词：

- 发布会感：像 Keynote 主舞台，不像传统汇报页。
- 大气：标题要极大，页面要敢留白，主体模块要成为舞台锚点。
- 聚焦：每页只讲一个主判断，其他内容都服务于这个判断。
- 温暖：用奶油、沙、陶土橙建立亲和感，避免冰冷科技蓝。
- 高端克制：少渐变、少盒子、少装饰，用比例和留白建立质感。
- 宣讲友好：远距离可读，正文不小于 18px，标题优先级绝对明显。

参考图风格提炼：

- 背景接近暖白奶油色。
- 主标题使用极粗黑色，带强烈信息冲击。
- 标签使用浅粉沙色胶囊，文字为陶土橙。
- 强调符号、勾选、图形中心使用陶土橙。
- 页面内容纵向居中，而不是从左上角开始堆。
- 默认背景使用纯色暖奶油底；渐变只能作为极低透明度的辅助光感，不作为主视觉。

## 2. 色彩系统

### 主色板

| Token | 颜色 | 用途 |
| --- | --- | --- |
| `--cream` | `#fbf7ef` | 全局背景，接近参考图的暖白 |
| `--cream-2` | `#fffaf3` | 卡片、浮层、亮面 |
| `--sand` | `#efe0cf` | 浅沙色块、章节背景、次级面 |
| `--sand-2` | `#ead4c3` | 胶囊标签、弱强调底 |
| `--clay` | `#c46f4f` | 主强调色，参考图里的陶土橙 |
| `--clay-dark` | `#a9553d` | 强调文字、按钮、进度条 |
| `--ink` | `#202020` | 超大标题和核心文字 |
| `--stone` | `#6f6a63` | 正文、说明文字 |
| `--muted` | `#9a9188` | 页码、辅助标签、弱线条 |
| `--sage` | `#7b8a78` | 少量辅助结构色，用于平衡全橙 |


### 背景策略

结论：默认使用纯色或近似纯色背景，更稳重、更高端。

推荐：

- 主背景使用 `#fbf7ef` 或 `#f8f1e8`。
- 可以叠加极淡纸感网格、噪点或单条舞台线，透明度控制在 `0.02-0.08`。
- 可以使用非常弱的径向光晕，但只能制造空气感，不能看起来像明显渐变海报。
- 发布会风格的视觉能量应该留给大标题、巨型编号、主视觉和关键转场。

避免：

- 大面积多色渐变背景。
- 紫蓝科技渐变。
- 过重的光斑、玻璃拟态、装饰 blob。
- 每页背景变化过大，导致风格不稳定。

### 色彩比例

- `70%` 奶油背景和暖沙面。
- `15%` 黑色/深灰文字与结构线。
- `10%` 陶土橙强调，包括标签、编号、关键词、进度、箭头、关键节点。
- `5%` 鼠尾草绿或灰褐，用于辅助信息。

### 使用规则

- 陶土橙必须“看得见”，不要只用于一个小点。每页至少出现 2 个橙色元素，如 eyebrow、关键词、编号、箭头、进度。
- 不建议整页大面积纯橙，除非是章节页或转场页。
- 黑色标题必须足够粗，不能用普通正文黑替代。
- 背景可以有极淡网格、沙粒、圆形光晕，但透明度要低于 `0.12`。

## 3. 字体与字号

### 字体建议

中文优先：

```css
font-family: "Alibaba PuHuiTi", "PingFang SC", "Microsoft YaHei", "Noto Sans CJK SC", sans-serif;
```

如果环境没有阿里巴巴普惠体，系统会回退到苹方或微软雅黑。

### 字号层级

以 16:9、1280x720 为基准：

| 类型 | 字号 | 字重 | 行高 |
| --- | --- | --- | --- |
| 封面标题 | `76-96px` | `900` | `1.02-1.08` |
| 章节标题 | `64-84px` | `900` | `1.05-1.12` |
| 内容页标题 | `46-64px` | `850-900` | `1.1-1.18` |
| 主张说明 | `24-30px` | `600-750` | `1.45-1.6` |
| 卡片标题 | `24-30px` | `800-900` | `1.18-1.28` |
| 卡片正文 | `18-22px` | `500-650` | `1.45-1.65` |
| 图表标签 | `17-22px` | `650-800` | `1.25-1.45` |
| 眉标/胶囊 | `14-17px` | `750-850` | `1` |

禁止：

- 投影正文低于 `18px`。
- 一页塞 8 行以上正文段落。
- 标题和正文差异太小。

## 4. HTML 舞台骨架

所有页面必须使用三层结构，避免内容散落在 `.slide` 里：

```html
<section class="slide layout-cover">
  <div class="slide-stage">
    <header class="title-block">
      <div class="eyebrow">Claude Code Skill</div>
      <h1>让 AI 生成真正能讲的 HTML PPT</h1>
      <p class="claim">不是把文字排进卡片，而是把观点组织成舞台。</p>
    </header>
    <div class="proof-object">
      <!-- 卡片、流程、图表、图片、时间轴等主体对象 -->
    </div>
  </div>
</section>
```

基础尺寸：

```css
.slide {
  width: 100vw;
  height: 100vh;
  min-width: 960px;
  min-height: 540px;
  padding: 6vh 7vw 9vh;
}

.slide-stage {
  width: min(1120px, 100%);
  min-height: 68vh;
  display: grid;
  align-content: center;
}

.proof-object {
  width: 100%;
  margin: 34px auto 0;
}
```

布局判断：

- 封面、章节、结尾：内容组中心位于页面高度 `48%-52%`。
- 普通内容页：内容组中心位于 `50%-56%`。
- 技术/架构页：可以略高，但主体图必须延伸到下半屏。
- 底部保留至少 `72px` 安全区给页码、进度条和导航。
- 如果缩略图中任何内容贴近上下边界，视为版式失败。优先改为双列、压缩卡片间距、降低结论条高度或拆页，不要硬塞。
- 目录或长列表超过 5 条时默认使用双列；超过 8 条且每条较长时拆成两页。
- 底部进度条只承担导航反馈，不承担装饰功能：高度约 `2px`，低透明度，翻页按钮使用浅色半透明，不使用深色实心圆。
- 带章节编号的章节页不使用标题下横线，保持干净高级；非章节的一句话观点页可以保留居中横线作为停顿和强调。
- 封面主动画图标不能遮挡标题。标题很宽时，主图标应落在 `<section class="slide layout-cover">` 内部的右下开放区域，不能用 `fixed` 或负值跑出封面元素；右上轨道图标应更靠右上角并降低透明度，只做氛围。
- 带章节编号的章节页只允许序号、章节标题、副标题和淡色动态图标。不要在章节页下方继续塞卡片、列表、架构层、表格或时间轴；这些内容必须拆到下一页。

## 5. 通用组件

### 胶囊标签

适用：封面分类、章节编号、页面 eyebrow、重点主题。

规则：

- 背景 `#ead4c3` 或透明浅沙。
- 文本 `#c46f4f`。
- 圆角 `999px`。
- 内边距 `12px 24px`。
- 不超过 2 个并排主标签，避免像筛选器。

### 卡片

适用：三点价值、多角色、多能力、方案模块。

规则：

- 背景 `rgba(255,250,243,0.82)`。
- 边框 `1px solid rgba(105, 82, 61, 0.16)`。
- 圆角 `22px`，参考图里图标卡片的柔和感。
- 阴影 `0 22px 70px rgba(93, 61, 34, 0.10)`。
- 2 卡最小高度 `220px`。
- 3 卡最小高度 `230px`。
- 4 卡建议用 `2x2`，不要一行四个小盒子。

### 强调数字

适用：章节号、步骤号、指标、时间点。

规则：

- 数字使用陶土橙。
- 字号比同区域标题大 `1.2-1.8` 倍。
- 可以作为半透明大号背景数字，但透明度不超过 `0.10`。

### 线条和箭头

适用：流程、时间轴、架构连接。

规则：

- 主箭头使用 `--clay`。
- 次级连接使用 `rgba(32,32,32,0.18)`。
- 线条要直、短、少，不做复杂曲线迷宫。

## 6. 页面类型与布局要求

总原则：发布会风格优先使用巨型标题、分屏、横向跑道、主视觉、巨型编号、少量大信息带。只有当内容确实是并列清单时才使用卡片；不要把所有页面都做成传统“标题 + 三卡片”。

### 6.1 封面页 `layout-cover`

使用场景：

- 主题开场。
- 重要宣讲。
- 方案首次亮相。

结构：

- 顶部或标题上方：1-2 个胶囊标签。
- 中部左对齐：超大标题，2-3 行以内。
- 标题下方：讲者、日期、场景说明。
- 右下或中下：一个抽象图形、logo 卡片或关键词组。

要求：

- 标题必须占据页面宽度 `70%-90%`。
- 关键词可用陶土橙或局部下划线突出。
- 装饰图形不能抢标题。

推荐文案示例：

- `让 Agent 边干边记：把经验沉淀成可复用技能`
- `HTML PPT 不是网页截图，而是一套宣讲舞台系统`

### 6.2 目录页 `layout-agenda`

使用场景：

- 汇报结构预告。
- 章节导航。

结构：

- 左侧：大标题 `今天只讲四件事`。
- 右侧：3-5 个目录项，每项含编号、标题、说明。
- 当前章节可用陶土橙编号。

要求：

- 目录项不要做成很小的列表，要像“章节卡片”。
- 每项说明最多 1 行。
- 编号字号 `34-48px`。

### 6.3 章节页 `layout-section`

使用场景：

- 进入新章节。
- 重要转折。

结构：

- 大号章节号，半透明或实色陶土橙。
- 一句大标题。
- 一句短 claim。
- 可有 2-3 个关键词。

要求：

- 章节页要比内容页更空、更有冲击。
- 标题不超过 18 个中文字为佳。
- 背景可使用更明显的沙丘圆形或斜纹。

### 6.4 舞台信息页 `layout-principles` / 多卡片页 `layout-cards`

使用场景：

- 三个核心观点。
- 三类用户。
- 三种能力。
- 四个原则。

结构：

- 发布会优先使用横向信息带：大编号 + 大标题 + 一句解释。
- 如果必须并列展示 3-4 个对象，再使用大卡片。
- 每个对象必须有编号、关键词或视觉锚点。
- 底部可有一句收束结论。

要求：

- 不要默认三卡片；三卡片是最后选择，不是默认选择。
- 信息带高度建议 `110-150px`，让页面像发布会大屏。
- 3 卡一行时，每卡高度至少 `230px`。
- 卡片之间间距 `24-30px`。
- 卡片内不能塞长段落。

### 6.5 有图片页 `layout-image-split`

使用场景：

- 展示产品截图。
- 展示人物/场景。
- 展示参考图或案例。

结构：

- 左文右图或右文左图，比例 `46:54` 或 `42:58`。
- 文案区：标题、claim、2-3 个要点。
- 图片区：大圆角截图卡片，带柔和阴影。
- 图片上可有小标签标注重点。

要求：

- 图片必须足够大，占页面宽度至少 `42%`。
- 不要把图片塞进小卡片列表。
- 图片页的文字要更少，让观众看图。

### 6.6 时间轴页 `layout-timeline`

使用场景：

- 项目路线图。
- 能力成熟路径。
- 演进阶段。

结构：

- 横向主时间轴，3-5 个阶段。
- 每阶段：时间/阶段名 + 结果描述。
- 当前阶段使用陶土橙实心点或竖线。
- 底部一条“判断/原则”总结。

要求：

- 不要做成竖向小字列表。
- 时间轴应横跨页面 `80%-92%`。
- 每阶段正文最多 2 行。

### 6.7 对比页 `layout-contrast`

使用场景：

- 旧方式 vs 新方式。
- 工具能力 vs 业务结果。
- 普通生成 vs 宣讲级生成。

结构：

- 标题说明对比结论。
- 两个大面板左右对照。
- 左侧弱化，右侧用陶土橙强调。
- 中间可以有箭头或“转折点”标记。

要求：

- 两个面板高度一致。
- 右侧不是简单变橙，而是有明确“为什么更好”的证据。
- 每侧最多 3 个 bullet。

### 6.8 流程页 `layout-flow`

使用场景：

- AI 生成 PPT 的步骤。
- 从输入到输出的链路。
- 交付流程。

结构：

- 4-5 个横向步骤模块。
- 每步一个动词：理解、拆解、选型、生成、校验。
- 陶土橙箭头连接。
- 底部标出关键控制点。

要求：

- 流程模块等高。
- 每步标题不超过 6 个中文字。
- 箭头数量要少，方向明确。

### 6.9 架构页 `layout-system`

使用场景：

- 技能系统结构。
- Agent 工作流。
- 数据/模板/渲染关系。

结构：

- 左侧输入：用户材料、参考风格、约束。
- 中间核心：布局决策器、组件库、样式系统。
- 右侧输出：HTML deck、设计规范、QA 报告。
- 底部：稳定资产 vs 可变内容说明。

要求：

- 中间核心模块要最大、最亮。
- 输入输出不要比核心抢眼。
- 连接线要清楚显示数据流方向。
- 技术词可以出现，但必须配业务解释。

### 6.10 数据/指标页 `layout-metrics`

使用场景：

- 展示效果提升。
- 展示效率或覆盖度。
- 汇报阶段成果。

结构：

- 1 个大指标 + 2-3 个辅助指标。
- 大指标放左或居中，字号 `84-120px`。
- 指标说明要解释业务意义。

要求：

- 不要一页放超过 5 个数字。
- 数字要和结论绑定，不是孤立 KPI。

### 6.11 结尾页 `layout-closing`

使用场景：

- 总结。
- 行动号召。
- Q&A。

结构：

- 一句超大 closing thesis。
- 3 个 takeaway 或下一步。
- 底部放联系信息、项目名或日期。

要求：

- 不要只写 `谢谢`。
- 结尾应该留下一个判断，例如：`好的 PPT 技能，先决定怎么讲，再决定怎么排。`

## 7. 内容生成规则

每页生成前先问自己 4 个问题：

- 这一页的主判断是什么？
- 观众看完 5 秒后应该记住哪句话？
- 证明这个判断的对象是什么：卡片、图片、流程、时间轴、架构还是指标？
- 这一页和上一页的版式是否重复？

推荐页面节奏：

1. 封面：大题目。
2. 目录：讲什么。
3. 章节：提出问题。
4. 对比/案例：说明痛点。
5. 多卡片：抽象原则。
6. 流程：说明怎么做。
7. 架构：说明系统如何支撑。
8. 时间轴：说明如何落地。
9. 结尾：收束判断。

## 8. Claude Code 实现提示词

可以把下面这段作为后续开发 skill 的核心要求：

```text
请生成 16:9 HTML PPT，风格是高端发布会 / Keynote 演讲，而不是传统汇报 PPT。视觉为奶油沙丘色：默认纯色暖白背景 #fbf7ef，陶土橙 #c46f4f，大黑标题 #202020，浅沙辅助 #ead8c8。不要使用明显大面积渐变，最多使用极低透明度纸感纹理或细线。每页必须使用 .slide > .slide-stage > .title-block + .proof-object 的结构。内容块必须垂直居中，不能从左上角堆叠。标题要极大，正文投影字号不小于 18px。根据内容选择封面、目录、章节、舞台信息带、图片主视觉、时间轴跑道、左右分屏、系统主视觉、结尾等布局。每页只表达一个主判断，并用一个大主体对象承载信息。陶土橙每页都要有足够存在感，用于编号、关键词、箭头、进度、标签或关键节点。避免传统 PPT 的小卡片堆叠、密集长段落、网页式组件、过度装饰和明显渐变海报感。
```

## 9. QA 检查清单

- 标题是否足够大，5 米外能不能读到？
- 这一页有没有唯一主判断？
- 主体对象是否位于舞台中心，而不是左上角？
- 卡片/图表是否占据页面中下部，避免右下空？
- 正文是否不小于 `18px`？
- 陶土橙是否形成风格存在感？
- 页脚、进度、导航是否没有压到内容？
- 连续两页是否避免完全同构？
- 图片页的图片是否足够大？
- 技术页是否有“业务解释”，不是只堆模块名？


## 10. 发布会版式增强规则

### 舞台安全区

- 16:9 页面不要把内容顶到上下边缘。推荐使用 `--stage-top: clamp(52px, 8.2vh, 92px)` 和 `--stage-bottom: clamp(86px, 11vh, 128px)`。
- 内容舞台宽度使用 `min(1180px, calc(100vw - var(--stage-x) * 2))`，高度使用 `calc(100vh - top - bottom)`。
- 字号、主视觉、面板高度都使用 `clamp()`，同时参考 `vw` 和 `vh`，避免宽屏或低高屏时溢出。
- 页脚、进度条和导航属于舞台外信息，不应压到主体内容。

### 主色使用比例

- `#a8553d` 不只是点缀色，而是发布会识别色。
- 每页至少在 3 类元素中使用它：关键词、编号、眉标线、进度、路径线、当前节点、图标或重点面。
- 推荐占视觉面积约 `12%-18%`，但避免整页铺满，保持奶油底的高级感。

### 新增叙事版式

- `layout-statement`：一句话大判断页，用于抛出关键观点；居中大标题 + 一条陶土色细线。
- `layout-metrics`：数据发布页，用于给证据；一个巨型数字 + 右侧 2-3 条解释。
- `layout-story`：案例故事页，用于讲转折；左侧巨型章节/案例号，右侧 A/B/C 三段叙事。
- `layout-hero-visual`：图片或产品主视觉页，用于展示截图、产品或概念装置；图片占半屏以上。
- `layout-runway`：横向跑道页，用于路线图、流程和发布节奏。

### 首页高级感

- 首页可以有淡色动效，但必须低对比、低饱和、慢速。
- 推荐元素：细线轨道、半透明抽象符号、轻微旋转的陶土色图标、缓慢浮动的卡片。
- 避免元素：粒子爆炸、强光效、多色渐变、快速旋转、大面积高饱和图形。

## 11. 全量布局模板清单

开发 skill 时必须覆盖以下页面类型，并按内容语义选择，而不是随机套模板。

### 基础通用布局

- `text-only`：纯文字页，标题 + 正文段落 + 要点罗列。
- `text-image-right`：左文右图，先讲观点，再给证据或产品画面。
- `image-text-right`：左图右文，先看画面，再解释意义。
- `image-top-text-bottom`：上图下文，先给场景，再收束解释。
- `text-top-image-bottom`：下图上文，先给结论，再用宽图证明。
- `full-image`：单大图页，全屏配图 + 少量标题文案。
- `icon-text`：图标文字页，图标搭配短句说明。

### 并列板块布局

- `two-cards`：双卡片布局，左右两分栏对等展示。
- `three-cards`：三卡片布局，三等分板块陈列内容。
- `grid-four`：四宫格布局，四块内容规整排布。
- `multi-column-list`：多列列表，纵向多栏信息罗列。

### 对比分析布局

- `left-right-compare`：左右对比页，两项内容差异对照。
- `pros-cons`：优劣对比页，优势 / 短板分项比对。
- `before-after`：新旧 / 前后对比，状态、方案更替对比。
- `comparison-matrix`：多维对比，多维度指标横向比对。

### 流程时序布局

- `horizontal-timeline`：横向时间轴，事件按时间先后排布。
- `vertical-timeline`：纵向时间轴，阶段进程竖向展示。
- `step-flow`：步骤流程图，分步操作、执行流程。
- `cycle-loop`：循环闭环图，往复循环类内容。

### 层级逻辑布局

- `main-branches`：总分结构页，总观点 + 分支要点。
- `tree-hierarchy`：层级树状页，从属、上下级关系。
- `radial-hub`：中心辐射页，核心内容向外延伸。
- `nested-hierarchy`：嵌套层级页，包含归属关系展示。

### 数据图表布局

- `bar-chart`：柱状图页，数据高低对比。
- `pie-chart`：饼图页，占比份额展示。
- `line-chart`：折线图页，趋势变化呈现。
- `combo-chart`：组合图表页，多类数据融合展示。

### 专项主题布局

- `conclusion`：观点结论页，核心看法总结输出。
- `problem-solution`：问题对策页，问题对应解决方案。
- `goal-plan`：目标规划页，既定目标 + 落地计划。
- `agenda-index`：目录索引页，全篇内容大纲梳理。
- `thanks-closing`：致谢封底页，收尾简约版式。
- `subtitle-band-cards`：强副标题提示条 + 三张卡牌 + 底部结论。
- `delivery-flow-cards`：三阶段交付流程卡牌 + 箭头 + 角色转变结论。

### 去重规则

- 目录索引页只保留在开头或章节导航处，不要把普通内容页做成目录式列表。
- 观点结论页、目录页、致谢页都可以很简洁，但语义不同，不能混用。
- 如果两个页面都是“编号 + 短句列表”，必须重新判断是否应该改成图文、对比、时间轴、层级或图表。


## 12. 副标题强调与彩色卡牌规则

### 强副标题提示条

适用场景：副标题本身是重要判断、背景条件、阶段说明或解释性结论时。

结构：

- 顶部：kicker + 超大标题，标题中关键词用 `#a8553d` 高亮。
- 标题下方：浅陶土背景提示条，左侧 6-8px 陶土色竖线。
- 提示条文字字号建议 `22-31px`，其中关键词加粗并使用 `#a8553d`。
- 提示条不应太长，最好是一句话解释当前页的上下文。

推荐布局：

- `subtitle-band-cards`：强副标题提示条 + 三张卡牌 + 底部结论。
- `delivery-flow-cards`：强标题 + 三阶段流程卡牌 + 箭头 + 底部角色转变结论。

### 标题关键词高亮

- 内容页标题中必须识别 1 个关键词或关键短语，使用陶土色 `#a8553d`。
- 不要整句标题都变色，只突出决策词、转折词、对象词或结果词。
- 中文大标题推荐黑色占主体，陶土色占 `15%-35%` 字符长度。

### 带颜色卡牌

- 当多个卡牌中有“当前阶段、推荐方案、核心角色、关键节点”时，使用浅陶土色卡牌突出。
- 彩色卡牌背景建议 `rgba(168,85,61,.095)`，边框 `rgba(168,85,61,.30)`。
- 一组 3 张卡里通常只突出 1 张；最多突出 2 张，否则失去重点。
- 彩色卡牌不等于强饱和色块，仍然要保持奶油沙丘风格的克制感。


### 章节页与结论页淡色动效

- 章节页、观点结论页、致谢页可以使用低透明度动态图标作为气氛层。
- 推荐符号：`✦`、`◇`、`⌁`、细线圆环、虚线轨道。
- 颜色使用 `rgba(168,85,61,.05-.30)`，不要超过正文视觉权重。
- 动效要慢：`7s-60s`，只做轻微浮动或缓慢旋转。
- 结论页的副标题和横线必须居中，使用 `margin-left:auto; margin-right:auto; text-align:center`。
- 纯文字页要避免正文和要点挤在一起，推荐左右分区或提示条承载正文。

## 13. 封面与封底信息区

### 作者和时间标签

封面和封底都需要预留作者与时间信息区，避免只在正文或页脚里出现。

推荐结构：

```html
<div class="author-meta" aria-label="作者和时间">
  <span class="meta-tag">作者 · Sun</span>
  <span class="meta-tag">时间 · 2026.05.23</span>
</div>
```

样式规则：

- 标签使用浅陶土背景：`rgba(168,85,61,.085)`。
- 边框使用：`rgba(168,85,61,.16)`。
- 文字使用主色：`#a8553d`。
- 封面信息区放在主标题说明下方，左对齐即可。
- 封底信息区放在结论横线下方，并整体居中。
- 不要在封面同时出现过多胶囊标签；如果已有作者/时间，移除类似 `Dune Keynote`、`Layout Library` 的装饰标签，避免信息层级冲突。

### 封底动效与首页呼应

封底不是简单“谢谢”，需要有轻微仪式感，并与首页主视觉呼应。

推荐规则：

- 复用首页的抽象放射图标作为封底视觉锚点，例如 `thanks-mark`。
- 封底图标颜色可以比背景点缀更深，建议 `rgba(168,85,61,.72-.88)`。
- 保留低透明度轨道、虚线圆环或轻微浮动符号，形成收束感。
- 可以加入 2-3 个微弱脉冲光点，但不能干扰标题阅读。
- 封底所有文字必须居中，包括 kicker、标题、说明、横线、作者和时间标签。
- 封底横线使用 `margin-left:auto; margin-right:auto`，不要出现偏左。

### 纯文字页清洁排版

纯文字页容易显得乱，默认不要做成长段落堆叠。

推荐：

- 左侧：标题、claim、淡色动效符号。
- 右侧：正文提示条或浅色信息面板。
- 正文面板使用左侧陶土色竖线，承载一段主说明和最多 3 条要点。
- 如果正文超过 120 个中文字，应拆成两页或改为图文/总分结构。
